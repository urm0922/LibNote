## LibNoteとは

LibNoteは、複雑な図書館業務に関するマニュアル・FAQ・対応ノウハウを一元管理し、どんな人でも、必要な情報にすばやくアクセスできるようにする業務支援アプリです。図書館に限らず、拠点を複数持つ現場スタッフの業務知識を標準化するアプリとして活用できます。


## 制作背景

きっかけは私の業務経験です。
いくつもの分館(拠点)を持つ大規模図書館では、正規職員・委託スタッフ・新人スタッフなど業務経験やITリテラシーが多様な人員体制で均一のサービス品質を求められます。しかしながら、ふたを開けてみると、重要な業務情報にも関わらずスタッフ間に適切に共有されていなかったり、担当者や拠点によって利用者の対応にばらつきが見られるということが多々ありました。また、従来の紙やPDF中心のマニュアルは、年々文量が肥大化し、必要な情報を探しづらくうまく活用されていないという課題がありました。

LibNoteは、こうした現場課題のもとに、業務ナレッジを一元的にわかりやすく整理・共有することを目的に作成しました。


## 問題点
- 中央館と分館との情報格差があり、業務の情報が適切に共有されていない。
- 紙やPDFの業務マニュアルは、業務が複雑化して文量が多く、読むのに抵抗がある。堅苦しい表現で難解である。
- 業務上の不明点は結果的に中央館スタッフに問い合わせることになり、同じ質問のやりとりが組織全体の業務量を増加させる。
- 図書館は基本的に土日祝日問わずサービス提供をしているので研修の機会が少ない。

## 解決策
- スタッフ間の業務情報のばらつきを減らし、問い合わせ対応、研修、マニュアル更新を一元化する
- 生成Aiで情報を整理してマニュアルを読みやすくする
- スタッフの問い合わせからFAQやナレッジ記事を作成し、業務情報を蓄積する
- ナレッジ記事から研修用クイズを自動生成する

## 想定ユーザー
分館(拠点)を複数持つ大規模図書館(サービス提供施設)のスタッフを想定しています
|ロール|具体例|役割|
|:---|:---|:---|
|委託スタッフ|分館スタッフ・窓口職員・新人スタッフ|業務上の不明点を調べたい|
|直営スタッフ|中央館スタッフ・業務担当者・承認者|業務担当として業務上の不明点を回答|
|管理者|システム担当・承認者|カテゴリ管理・記事承認・ユーザー管理・AI下書き確認|

## 機能一覧

### ユーザー管理機能
- ユーザー登録
‐ ログイン・ログアウト
- ユーザー情報編集
- 権限管理

### 問い合わせ投稿機能
- 業務上の疑問・問い合わせを投稿
- タイトル、本文、カテゴリ、関連施設などを登録
- 投稿者情報の表示
- 投稿日時の表示
- 投稿内容の編集・削除

### 回答・コメント機能
- 投稿された問い合わせに対する回答
- 複数スタッフによる情報共有
- 回答履歴の確認

### ナレッジ化機能
- 問い合わせと回答をもとにナレッジ記事を作成
- 承認済みの内容を正式なナレッジとして登録
- よくある問い合わせをFAQとして蓄積
- 現場対応の判断基準を共有

### 承認ワークフロー機能
- 投稿内容を確認中・承認済み・差し戻しなどのステータスで管理
- 管理者または承認担当者による内容確認
- 未確認情報と正式ナレッジを区別
- 誤った情報がそのまま共有されるリスクを軽減

### カテゴリ管理機能
- 業務分野ごとにナレッジを分類
　(利用者対応・貸出、返却など…)

### 検索機能
- キーワード検索
- カテゴリ検索
- ステータス別検索
- よく使われる記事の検索
- 過去の問い合わせ・回答の参照

### マニュアル連携機能
- 業務マニュアルをもとに初期ナレッジを作成
- マニュアルの内容を踏まえて、定められた日数などは共通ルールとして管理
- 管理者・担当者の確認と承認を経て公開
- マニュアルの改定時には、再構成されたナレッジを確認し、既存ナレッジとの差分を更新

### 生成AI想定機能
- マニュアルや業務資料をもとに、ナレッジ下書きを生成
- 問い合わせ内容に関連するマニュアル箇所の提示
- 回答作成時の参考情報の引用に利用
- 生成AIの回答をそのまま公開せず、必ず人による確認・承認をおこなう設計
