## LibNoteとは

LibNoteは、複雑な図書館業務に関するマニュアル・FAQ・対応ノウハウを一元管理し、どんな人でも、必要な情報にすばやくアクセスできるようにする業務支援アプリです。図書館に限らず、拠点を複数持つ現場スタッフの業務知識を標準化するアプリとして活用できます。


## 制作背景

きっかけは私の業務経験です。
いくつもの分館(拠点)を持つ大規模図書館では、正規職員・委託スタッフ・新人スタッフなど業務経験やITリテラシーが多様な人員体制で均一のサービス品質を求められます。しかしながら実際の現場では、重要な業務情報がスタッフ間で十分に共有されていなかったり、担当者や拠点によって利用者対応にばらつきが生じたりする場面がありました。
また、従来の紙やPDF中心のマニュアルは、年々文量が肥大化し、必要な情報を探しづらくうまく活用されていないという課題がありました。

LibNoteは、こうした現場課題のもとに、業務ナレッジを一元的にわかりやすく整理・共有することを目的として開発中のアプリです。


## 問題点
- 中央館と分館との情報格差があり、業務の情報が適切に共有されていない。
- 紙やPDFの業務マニュアルは、業務の複雑化に伴って文量が増え、必要な情報が探しにくくなりがちである。表現が堅く、経験の浅いスタッフには理解しづらい場面がある。
- 業務上の不明点は結果的に中央館に問い合わせることになり、同じ質問のやりとりが組織全体の業務量を増加させる。
- 図書館は基本的に土日祝日問わずサービス提供をしているので研修の機会が少ない。


## 解決策
- 業務ナレッジを一元管理し、全拠点から同じ情報にアクセスできるようにする。
- カテゴリ・検索・FAQ化により必要な情報を見つけやすくする
- スタッフの問い合わせからFAQやナレッジ記事を作成し、再利用する
- ナレッジ記事やクイズ機能により自己学習しやすくする


## 想定ユーザー
分館(拠点)を複数持つ大規模図書館(サービス提供施設)のスタッフを想定しています
|ロール|具体例|役割|
|:---|:---|:---|
|委託スタッフ|分館スタッフ・窓口職員・新人スタッフ|業務上の不明点を調べたい|
|直営スタッフ|中央館スタッフ・業務担当者・承認者|業務担当として業務上の不明点を回答|
|管理者|システム担当・承認者|カテゴリ管理・記事承認・ユーザー管理・AI下書き確認|


## アプリの使用イメージ
1.分館スタッフが業務中に判断に迷った内容を問い合わせとして投稿する
2.中央館スタッフや業務担当者、管理者が回答する
3.回答内容をもとに、管理者がナレッジ記事として整理する
4.承認されたナレッジが全スタッフに共有される
5.他のスタッフが同じ疑問を持った際に、検索して自己解決できる


## 機能について

### 以下の機能を中心に実装予定
- ユーザー登録・ログイン機能
- 問い合わせ投稿のCRUD機能
- 回答・コメント機能
- カテゴリ機能
- 検索機能
- ステータス管理
- 管理者による承認機能
- ナレッジ一覧表示
- ナレッジ詳細表示
- レスポンシブ対応
- 落ち着いたイメージで茶色・ベージュ基調のUI

### 生成AI活用想定(将来的に広げるもの)
- マニュアルや業務資料をもとに、ナレッジ下書きを生成
- 関連マニュアル箇所の提示
- 回答作成時の参考情報提示
- 生成AIの回答をそのまま公開せず、必ず人による確認・承認をおこなう
